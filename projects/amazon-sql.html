Amazon USA Sales Analysis Project using SQL

Difficulty level: Advanced  |  GitHub Link

Project Overview

I have worked on analyzing a dataset of Amazon USA sales to uncover trends, top-performing categories, and actionable business insights. The goal was to use advanced SQL techniques to explore a realistic ecommerce sales dataset and answer key business questions.

Database Setup & Steps

The project involved creating a structured database schema, loading the raw data, and then running a series of queries that progressively increased in complexity — from basic aggregations through to window functions, CTEs, and subqueries.

Key SQL Techniques Used

  • Aggregate functions (SUM, COUNT, AVG)
  • GROUP BY with HAVING clauses
  • Window functions (ROW_NUMBER, RANK, LAG)
  • Common Table Expressions (CTEs)
  • Subqueries and nested SELECT statements
  • JOIN operations across multiple tables
  • Date functions for time-series analysis

Sample Queries

-- Top 10 best-selling products by revenue
SELECT
    product_name,
    category,
    SUM(sale_price * quantity) AS total_revenue,
    COUNT(order_id) AS total_orders
FROM amazon_sales
GROUP BY product_name, category
ORDER BY total_revenue DESC
LIMIT 10;
-- Month-over-month revenue growth
WITH monthly_revenue AS (
    SELECT
        DATE_TRUNC('month', order_date) AS month,
        SUM(sale_price * quantity) AS revenue
    FROM amazon_sales
    GROUP BY 1
)
SELECT
    month,
    revenue,
    LAG(revenue) OVER (ORDER BY month) AS prev_month,
    ROUND(
        (revenue - LAG(revenue) OVER (ORDER BY month))
        / LAG(revenue) OVER (ORDER BY month) * 100, 2
    ) AS growth_pct
FROM monthly_revenue
ORDER BY month;

Key Findings

  • Electronics and Home & Kitchen were the top revenue-generating categories.
  • Q4 showed consistent revenue spikes driven by holiday season demand.
  • A small percentage of products (top 10%) accounted for over 60% of total revenue — a classic Pareto distribution.
  • Repeat customers showed significantly higher average order values than first-time buyers.

Conclusions

This project demonstrated how raw transactional data can be transformed into business intelligence using structured SQL queries. Advanced window functions were particularly powerful for identifying trends over time without collapsing the underlying row-level data.
